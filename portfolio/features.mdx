## File tree component

The file tree is the primary navigation mechanism, displaying your portfolio content in a hierarchical structure that developers instantly recognize from their code editors.

### Structure

The file tree supports:

- **Files** - Individual content items (about, projects, etc.)
- **Folders** - Containers that group related content
- **Nested folders** - Unlimited nesting depth for complex organization
- **Icons** - Visual indicators for file types

### Interaction

Users interact with the file tree through:

| Action | Behavior |
|--------|----------|
| Click file | Opens file content in a new tab |
| Click folder | Expands/collapses folder contents |
| Hover | Visual feedback indicating clickable elements |

### Customization

The file tree appearance can be customized through:

- Icon selection for different file types
- Custom colors for folders and file types
- Indentation and spacing adjustments
- Font and typography settings

## Tabbed interface

The tab system provides a familiar multi-document interface:

### Tab behavior

- **Opening tabs** - Clicking a file creates a new tab
- **Switching tabs** - Click any tab to view its content
- **Closing tabs** - Click the X button on a tab
- **Active indicator** - Current tab is visually distinct
- **Minimum tabs** - At least one tab always remains open

### Tab state

The tab system maintains:

```typescript
interface Tab {
  id: string;      // Unique identifier
  name: string;    // Display name (filename)
  content: string; // Markdown content
}
```

### Visual design

Tabs match the IDE aesthetic:

- Connected appearance with main content area
- Active tab appears "raised" or highlighted
- Inactive tabs have subtle background difference
- Close button appears on hover for non-essential tabs
- Tab overflow scrolls horizontally on narrow screens

## Project images

Projects can include image galleries for screenshots and visuals:

### Image display

Images are displayed in a gallery format:

- Thumbnail grid layout
- Consistent sizing and spacing
- Responsive to container width
- Lazy loading for performance

### Image modal

Clicking an image opens a full-size view:

- Centered modal overlay
- Full resolution image display
- Click outside or X to close
- Keyboard navigation (Escape to close)

### Image optimization

For best performance:

- Use appropriate image formats (WebP, PNG, JPEG)
- Compress images before adding
- Provide responsive sizes where possible
- Use CDN hosting for production

## Responsive behavior

The portfolio adapts to different screen sizes:

### Desktop (1024px+)

- Full sidebar always visible
- Multiple tabs comfortable
- Optimal viewing experience

### Tablet (768px - 1023px)

- Sidebar may be collapsible
- Tab bar scrolls horizontally
- Touch-friendly interactions

### Mobile (< 768px)

- Sidebar hidden by default
- Menu button reveals sidebar as overlay
- Single-column layout
- Simplified navigation

### Mobile sidebar

On mobile devices:

1. Sidebar is hidden initially
2. Menu button in top bar toggles visibility
3. Overlay covers main content
4. Clicking outside closes sidebar
5. Selecting a file closes sidebar and opens content

## Extending the portfolio

### Adding new file types

To support new content types:

1. Define the content structure in `portfolio-data.ts`
2. Add rendering logic in `markdown-renderer.tsx`
3. Update file tree icons if needed
4. Style the new content type appropriately

### Integrating external data

The portfolio can be extended to fetch content from:

- CMS platforms (Contentful, Sanity)
- GitHub repositories (auto-generate from READMEs)
- Notion databases
- Custom APIs

### Adding interactivity

Potential interactive additions:

- Contact form with email integration
- Live code playgrounds
- Interactive demos embedded in content
- Analytics and visitor tracking
- Comments or guestbook
